# execution-mode-guide

実行モード `strict` / `balance` / `speed` の共通選択基準です。

## 判定軸

- リスク: 失敗時の影響が高いか
- 影響度: 影響範囲が広いか（機能横断、組織横断）
- 期限: 期限が厳しく、迅速性を優先する必要があるか

## 選択マトリクス

| 条件 | 推奨モード |
| --- | --- |
| 高リスク かつ 高影響度 | strict |
| 中リスク または 中影響度 | balance |
| 低リスク かつ 低影響度 かつ 短納期 | speed |

## モード比較

| モード | 証跡量 | 調査深度 | 合意形成の粒度 |
| --- | --- | --- | --- |
| strict | 最大 | 深い（再探索含む） | 詳細（判定根拠を明文化） |
| balance | 中 | 標準 | 必須判断点を記録 |
| speed | 最小必須 | 浅め | 要点のみ記録 |

## 非推奨ケース

- strict 非推奨: 低リスクの軽微修正で、期限が極端に短い場合
- balance 非推奨: 監査対象で厳密な証跡が必要な場合（strict を推奨）
- speed 非推奨: クリティカル機能、不具合再発対応、全社標準化に関わる変更

## 運用ルール

- モード選択理由を冒頭で明示する
- 途中でモード変更する場合は理由をログへ追記する
- 迷う場合は `balance` を初期値とし、必要に応じて `strict` へ引き上げる
