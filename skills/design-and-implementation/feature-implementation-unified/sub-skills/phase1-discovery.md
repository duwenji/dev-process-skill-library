# Phase 1 Sub-Skill: 要求整理・方式候補化（段階1-6）

## 概要
Phase 1では、要求を明確化し、現行との差分を分析して方式案を提示します。

## 責務
- AI: 差分分析、フロー生成、方式案提示
- 開発者: 要求入力、方式案比較、承認判断

## 段階1-2の実施ポイント
1. 対象モジュール、関連DDL、運用制約を特定
2. 要件を機能要件/非機能要件へ分解
3. 受入条件（判定可能な条件）を定義

## 段階3の入力テンプレート
```markdown
## 段階3: 開発者が要求を正式入力（日時）

### タイトル
[Module] [Feature/Change Summary]

### 背景
- [業務背景]

### スコープ
- [対象機能]

### 非スコープ
- [対象外]

### 受入条件
1. [条件1]
2. [条件2]

### 制約
- [納期/性能/互換性]
```

## 段階4-6の出力
- 差分分析レポート
- 現行/改修後フロー図
- 方式案比較表（最低3案）
