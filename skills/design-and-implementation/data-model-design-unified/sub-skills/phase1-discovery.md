# Phase 1: 要件整理・概念モデル抽出

## 目的
要件からエンティティ候補、主要属性、関係候補を抽出し、モデル化の論点を明確にする。

## 実施ポイント
1. 要件IDごとに必要データを整理する
2. エンティティ候補と値オブジェクト候補を分類する
3. 関係（1:1, 1:N, N:M）を仮定し、根拠を残す
4. 制約候補（一意性、必須、参照整合）を列挙する

## 段階3 入力テンプレート
```markdown
## 背景
- 

## スコープ
- 

## 非スコープ
- 

## 要件ID一覧
- REQ-001: 
- REQ-002: 

## エンティティ候補
- 

## データ制約
- 

## 性能要件
- 
```

## 段階4-6の出力
- 概念モデル候補一覧
- 概念 ERD（Mermaid）
- モデル案比較表（最低3案）
