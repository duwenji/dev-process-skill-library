# Intake: 知識収集

## 目的

ユーザが持つ暗黙知（ノウハウ、判断基準、失敗事例、手順）を、構造化に必要な粒度でチャットから引き出す。

## 実施ポイント

1. AI が収集テンプレートを提示し、ユーザの自由入力を促す
2. 不足している項目（背景・再現条件・適用範囲）を追加質問で補完する
3. 機密区分（公開可 / 限定公開 / 匿名化必須 / 記録不可）をユーザと確定する
4. 収集完了の合図（「以上です」「ここまで」等）でステージ完了とする

## 収集テンプレート

```
背景・状況:
課題・問題:
対処方法・解決策:
成功パターン:
失敗パターン・注意点:
対象領域・適用範囲:
再現条件:
測定・確認方法:
機密区分: （公開可 / 限定公開 / 匿名化必須 / 記録不可）
```

## 補完質問の例

- 「この知識は何人くらいのチームで使えますか？」
- 「この手順が失敗するのはどんな条件のときですか？」
- 「似たような知識が既存 Skill にありそうですか？」
