# テストケース CSV テンプレート

本テンプレートは、Markdown正本のテストチェック項目を CSV 化し、Excel で確認・更新するための標準フォーマットです。

## 運用方針
- 正本: Markdown（Phase 3 ドキュメント）
- 補助: CSV（Excel 確認、レビュー、集計）
- 反映順: Markdown -> CSV -> Excel確認 -> Markdownへ差分反映

## CSV ヘッダ（固定）

```csv
TRK,分類,テストケース名,前提条件,入力値,期待結果,実施方法,実施手順,実行ステータス,実行者,備考
```

## 値ルール
- 実施方法: 自動 / 手動 / ダミー実装
- 実行ステータス: 未実施 / 実施中 / 合格 / 不合格 / 保留 / スキップ
- TRK: 一意ID（再利用しない）

## サンプル

```csv
TRK,分類,テストケース名,前提条件,入力値,期待結果,実施方法,実施手順,実行ステータス,実行者,備考
TRK-001,正常系,全フィールド有効値で処理実行,有効データが存在,DisplayName=テストユーザー;RecordId=ID-001,正常処理される,自動,dotnet test --filter TRK-001,合格,AI,既存回帰対象
TRK-002,境界値,DISPLAY_NAME=NULL,DISPLAY_NAME が NULL,DisplayName=NULL,既定値を適用し継続,自動,NULL入力モックを作成して Assert,合格,AI,GetStringOrNull 動作確認
TRK-007,異常系,DB接続エラー時の処理,SqlException を模擬,SqlException 発生,例外伝播とエラーログ出力,ダミー実装,Mock で SqlException を投出してログ検証,未実施,AI,実DB障害は模擬
TRK-008,統合,連続処理の安定性確認,テスト環境で連続送信可,100件連続送信,処理完了かつリークなし,手動,送信後に件数とメモリ使用量を照合,未実施,手動担当,ストレス観点
```

## Excel 利用手順
1. CSV を UTF-8 BOM で保存する。
2. Excel で開き、列順がヘッダと一致していることを確認する。
3. 実施後に `実行ステータス` と `実行者` を更新する。
4. ゲート判定前に Markdown 正本へ差分を反映する。

## 品質チェック
- 未実施のまま承認しない。
- 不合格/保留には備考で原因と次アクションを記載する。
- 集計前に重複TRKと空欄列をチェックする。
