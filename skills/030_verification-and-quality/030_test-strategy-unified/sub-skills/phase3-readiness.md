# Phase 3: 実行準備・検証確認

省略用語（RACI, KPI, ADR, DDL, SLO, QA, PM, TRK, EX）は [../../../shared-references/glossary.md](../../../shared-references/glossary.md) の『略語・日本語対応表』を参照してください。


## 目的
実行前提、データ準備、環境制約、証跡の取り方を確定する。

## 実施ポイント
1. テストデータ、環境、依存サービスの準備を確認する
2. 実行順序と担当を決める
3. 各チェック項目に実施方法と実施手順を記載する
4. Markdown 正本から CSV（Excel確認用）を出力し、列定義を固定する
5. 実行ステータス別・実施方法別・分類別の3軸集計を作成する
6. 実行できない項目には代替策を付ける
7. リリース判断に必要な証跡を定義する

## チェック項目の標準列

```markdown
| TRK | 分類 | テストケース名 | 前提条件 | 入力値 | 期待結果 | 実施方法 | 実施手順 | 実行ステータス | 実行者 | 備考 |
```

- 実施方法: 自動 / 手動 / ダミー実装
- 実行ステータス: 未実施 / 実施中 / 合格 / 不合格 / 保留 / スキップ

## CSV運用（Excel確認用）
1. Markdown表を CSV に変換（ヘッダ順は固定）
2. UTF-8 BOM で保存し、Excel で列崩れを確認
3. 実施後に `実行ステータス` と `実行者` を更新
4. ゲート判定前に Markdown 正本へ差分反映

## 集計運用（ゲート判定用）
- 実行ステータス別集計: 未実施、不合格、保留の件数を確認
- 実施方法別集計: 自動/手動/ダミー実装の件数と結果を確認
- 分類別集計: 正常系/境界値/異常系/統合/回帰の進捗を確認

## ゲート #2 / #3 の確認
- [ ] チェック項目が主要リスクを網羅する
- [ ] 全チェック項目に実施方法と実施手順が記載されている
- [ ] CSV（Excel確認用）と Markdown 正本の内容が一致している
- [ ] 3軸集計（ステータス/実施方法/分類）が作成されている
- [ ] 実行不可項目の扱いが決まっている
- [ ] 残余リスクが記録されている
