# Pilot 導入ガイド

## Pilot の目的

全社展開前に小規模チームで動作検証し、運用課題を先行して収集する。

## Pilot 手順

1. 対象チームと Skill を選定する（初回: XXX チーム × test-strategy-unified, requirements-refinement）
2. 期間: 4週間
3. 適用案件: 2件以上
4. レトロスペクティブで改善点を収集する
5. 全社展開可否を判定する

## 完了条件

- 2件の案件で Skill を適用済み
- レトロスペクティブの記録が governance/quarterly-review-log.md にある
- 全社展開判定が Approver によって承認されている

## 90日展開計画

- 0-30日: Pilot 実施、運用課題収集
- 31-60日: 改訂版を2チームへ横展開
- 61-90日: 全社標準化判断と運用版 1.0 確定
